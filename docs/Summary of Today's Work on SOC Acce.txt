Summary of Today's Work on SOC Access Management Tool
Enhancements & Fixes Implemented:
GUI Overhaul for Consistency & Functionality
Improved Check, Give, and Remove Access windows to maintain a consistent user experience.
Standardized the layout, logging, and execution process across all three functions.
Check Access Window
Now correctly prompts for SOC user LDAP credentials and the target user's LDAP username.
Calls individual system scripts (AWS, JIRA, PagerDuty, Slack, GitHub, Azure) to check access.
Displays real-time logs to show script execution.
No Mock Mode included, as this function only checks access and is non-disruptive.
Give & Remove Access Windows
Implemented Mock Mode Toggle (Enabled by Default) for safety.
Allows SOC users to enter credentials & target user info, then calls the appropriate scripts.
Added real-time logging to display script execution status and errors.
Separation Script (separations.py) Now Supports Both GUI & CLI
If run via CLI, prompts user for LDAP credentials & target username interactively.
If run via GUI, pulls inputs from the window & respects the Mock Mode setting.
Runs LDAP queries, logs execution, and removes access in parallel for efficiency.
Improved error handling to log LDAP failures and missing scripts.
Refactored soctool.py Main GUI
All functions now open dedicated action windows rather than running scripts in the background.
Global Mock Mode Toggle for Give & Remove Access (enabled by default).
Improved responsiveness by running script executions in separate threads.
Implemented real-time logging in pop-ups for better troubleshooting.
