📌 Summary of Recent Updates to the SOC Access Management Project

Since the last update, we made significant changes to the SOC Access Management Scripts based on feedback from Matt and Dean. These updates prioritize maintainability, CLI usability, RT ticket integration, and structured logging.

🔹 Key Changes Implemented
1️⃣ Refactored separation.py, giveaccess.py, and checkaccess.py for CLI-First Design
CLI Usability: The scripts are now designed to be executed via the command line, ensuring they work on the SOC workstation in Azure without dependency on a GUI.
Maintainability: Code is structured with clear variable names and logging to facilitate ease of understanding and future updates.
Optional GUI Support: A GUI interface is still available for those who prefer it.
2️⃣ Implemented RT Ticket Integration
Added an optional RT ticket number argument to all scripts.
When provided, the scripts automatically add a comment to the RT ticket with:
Check Access: Summary of the user's current access.
Grant Access: Systems the user was granted access to.
Remove Access: Systems the user was removed from.
Ensures documented change tracking within RT for accountability.
3️⃣ Logging Enhancements
Centralized Logging: All actions (checks, grants, removals) now log to SOC_Access_Log.txt.
Clear Execution Trail: Logs include timestamps, system names, and action summaries.
Error Handling: Improved handling for missing scripts, failed commands, and LDAP authentication issues.
4️⃣ Introduced rt_api.py for RT API Interactions
A new helper module (rt_api.py) handles RT ticket commenting.
Makes it easier to maintain RT-related functionality separately.
Standardizes RT API calls across all scripts.
5️⃣ Mock Mode for Grant & Remove Access
Added a mock mode toggle (--mock flag).
Allows safe testing without actually modifying access.
Enabled by default in the GUI, but can be disabled.
6️⃣ Improved System-Specific Script Execution
The scripts now dynamically locate and execute system-specific scripts (e.g., remove_aws.py, grant_slack.py, etc.).
If a system script is missing, a warning is logged instead of failure.
7️⃣ Updated GUI for Optional Use
GUI now includes:
RT Ticket Number Input for automatic RT ticket commenting.
Mock Mode Toggle (default enabled for grant/remove access).
GUI remains optional, ensuring CLI functionality remains primary.

🚀 Deploy the updated scripts for live use in the SOC environment.
