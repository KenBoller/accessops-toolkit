I’m working on a Python-based SOC Access Management Tool for Marchex’s internal SOC team. It’s designed to manage user access across internal systems via both a CLI and Tkinter GUI frontend, with LDAP authentication and audit logging.

Here's where the project currently stands:

✅ Project Goals
Allow SOC analysts to check, grant, and remove user access to internal systems.
Support both interactive CLI and GUI (Tkinter/X11) usage.
Require LDAP authentication before actions are allowed.
Include RT ticketing integration and mock mode for testing.
Be simple, portable, and self-contained — no unnecessary complexity like external microservices or modules.
📁 Current Code Structure
soctool.py – Tkinter GUI wrapper.
access_management.py – CLI backend for access operations (handles all business logic).
Utils/get_secrets.py – Loads secrets from a local JSON file.
Utils/rt_api.py – (Optional) for writing comments to RT tickets.
Systems/UIs/ – Folder of individual check/grant/remove scripts per system (e.g., check_jira.py, remove_grafana.py, etc.).
Logs/ – Centralized log storage for all actions.
